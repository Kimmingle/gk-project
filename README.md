# 🗣 사내 커뮤니티 (React Portfolio)

회사 구성원들과 함께 **React 및 웹 서비스 개발 역량을 학습하기 위한 사내 스터디 프로젝트**로 진행한  
**사내 커뮤니티 웹 애플리케이션**입니다.

실제 사내에서 사용 가능한 서비스를 목표로 하여  
**인증/인가, 권한 관리, CRUD, 관리자 기능** 등  
실무에서 자주 요구되는 기능을 중심으로 설계 및 구현했습니다.

---

## 📌 프로젝트 개요

- **프로젝트 목적**
  - 사내 스터디를 통해 React 기반 웹 서비스 개발 경험 축적
  - 실무에서 사용되는 구조와 패턴을 학습 및 적용
  - 익명 커뮤니티 특성을 고려한 권한 및 관리 기능 구현

- **프로젝트 성격**: 사내 스터디 학습용 프로젝트
- **개발 인원**: 스터디원 4명
- **담당 역할**: 프론트엔드 개발 (주요 기능 구현 및 구조 설계)

---

## 🛠 기술 스택

### Frontend
- React
- JavaScript (ES6+)
- React Router
- Axios

### State / Auth
- Context API / Redux
- 토큰 기반 인증 (JWT)

### Style
- CSS / SCSS / Styled-components

### Tool
- Git / GitHub
- REST API

---

## ✨ 주요 기능

### 🔐 인증 / 인가
- 로그인 / 회원가입
- 토큰 기반 인증 처리
- 로그인 상태 유지
- 인증 여부에 따른 페이지 접근 제어

---

### 🕶 익명 커뮤니티
- 게시글 익명 작성
- 게시글 목록 / 상세 조회
- 익명 댓글 작성
- 작성자 정보 비노출 처리

---

### 📝 게시판 (CRUD)
- 게시글 작성 / 수정 / 삭제
- 댓글 작성 / 삭제
- 본인 작성 글에 대한 수정/삭제 권한 제한

---

### 🔑 권한 관리
- USER / ADMIN 권한 분리
- 관리자 전용 기능 제공
- 일반 사용자는 익명 유지

---

### 🛡 관리자 기능
- 전체 게시글 관리
- 신고된 게시글 처리
- 부적절한 게시글 삭제
- 사용자 제재 기능

---

## 🧠 기술적 고민 & 학습 포인트

### 1️⃣ 익명성과 인증의 균형
- 화면에서는 사용자 정보 비노출
- 서버 단에서는 사용자 식별 가능하도록 설계
- 익명 커뮤니티에서의 책임성과 관리 포인트 학습

---

### 2️⃣ 권한 기반 접근 제어
- 라우터 레벨에서 접근 제어
- 관리자 페이지 분리
- 공통 권한 체크 로직으로 유지보수성 향상

---

### 3️⃣ 스터디 목적에 맞춘 구조 설계
- 기능 단위 컴포넌트 분리
- CRUD 공통 로직 재사용
- 팀원 간 코드 리뷰를 고려한 폴더 구조
